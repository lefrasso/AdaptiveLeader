# The Adaptive Leader — web app

The modern, statically-exported **Next.js** build of *The Adaptive Leader*: an
interactive, multilingual leadership guide with twenty-two chapters, four
appendices, five assessment tools, and a growth-mindset learning loop (quizzes,
reflections, spaced review, and a progress dashboard).

## Stack

- **Next.js 16** (App Router, `output: 'export'` — a fully static site)
- **React 19** + **TypeScript**
- **Tailwind CSS v4** + shadcn-style UI primitives (Radix)
- **next-intl** for i18n across 10 locales (RTL for Arabic)
- **Vitest** for the pure-logic unit tests
- **pnpm**

## Getting started

```bash
pnpm install
pnpm dev          # http://localhost:3000  (root redirects to /en)
```

Other scripts:

```bash
pnpm build        # static export to ./out  (postbuild writes the root redirect)
pnpm test         # vitest run
pnpm lint         # eslint
pnpm version:check # validate package.json as SemVer
```

The version in `package.json` is injected into the static build and displayed
in the About dialog. Releases use a matching `vX.Y.Z` Git tag; see the root
`CHANGELOG.md` for release notes.

## Project structure

```
src/
  app/[locale]/           # routes: home, chapters, appendices, assessments, cultural-tips
  components/
    assessments/          # self, leadership, communication, cultural tools
    learning/             # chapter quiz, practice, growth dashboard
    motion/               # reveal-on-scroll
    ui/                   # Radix-based primitives
  i18n/                   # next-intl routing / request / navigation
  lib/
    chapters/             # per-locale chapter JSON + loaders
    appendices/           # per-locale appendix JSON + loaders
    assessments/          # pure scoring logic (+ vitest tests)
    cultural/             # cultural-tips data
    prefs/                # preferences + consent store (localStorage)
    progress/             # learning-progress store + spaced-review scheduler
messages/                 # per-locale UI strings (next-intl)
scripts/                  # content pipeline (extract / expand / translate)
```

## Internationalisation

- UI strings live in `messages/<locale>.json`; long-form content (chapters,
  appendices) in `src/lib/**/<locale>.json`.
- **English-first**: new content is authored in English and other locales fall
  back to English until translated (`request.ts` deep-merges over `en`, and the
  chapter loader appends any English-only chapters a locale is missing).
- Content is generated by the scripts in `scripts/` — e.g. `extract-chapters`,
  `extract-i18n`, `extract-ui`, `expand-theory`, and `double-theory`.

## Themes

Four themes via `next-themes`: **Default**, **Retro**, **Dark & Modern**, and
**Copilot** (Microsoft 365 / Fluent palette). The choice is remembered locally.

## Local data & privacy

Nothing is sent to a server. A first-visit consent banner separates essential
preferences (theme, language) from opt-in learning progress. The **Privacy &
data** panel lets you toggle saving, export your data, or erase it. Progress
persists to `localStorage` only when consented.

## Deployment

Published to GitHub Pages by `.github/workflows/deploy.yml` (lint → test → build
→ deploy). The build sets `NEXT_PUBLIC_BASE_PATH=/AdaptiveLeader`; a postbuild
script writes `out/index.html` to redirect the root to a locale. Enable it via
**Settings → Pages → Source: GitHub Actions**.
